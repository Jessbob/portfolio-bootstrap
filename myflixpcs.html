myFlix

project case study


Overview

myFlix is a web app that allows users to see details about movies. The user can also create a profile and save movies to their favorites list.

Movie Details

  • Movie Title
  • Movie Description
  • Movie Cover Art
  • Genre
  • Director

Genre Details

  • Genre Name
  • Genre Description

Director Details

  • Name
  • Biography
  • Birthdate

Purpose

The myFlix movie API was an excellent project provided by CareerFoundry. Through this course, I learned how to create a full-stack web app using the MERN stack (MongoDb, Express, React, Nodejs). This project went through every process needed to create a database and transfer data to and from the React app with a REST API. I really enjoyed working on this project it brought a lot of fun things together from working with databases to coding modern web apps and working with enjoyable data like movies kept it entertaining.

Approach

The first step in this project was to get the backend setup. This process started with defining API endpoints to visualize the virtual routes that the data will travel. Once the endpoints were defined, Nodejs and Express created the routes for the server. I used mock data and Postman to test the different routes and ensure the server was responding appropriately. After the test proved successful, the real data was loaded into a non-relational database created with MongoDb. With the back-end finished we needed a front-end interface to access and modify the data. For the interface React was a great choice since its original intention was to create user interfaces, it also boasts an active developer community and robust set of tools. Setting up the interface started by building the main view that would control the subviews of the app.

The single-page app would consist of the following subviews

  • Login View- Allows the user to login to their profile
  • Registration View- Allows new users to register for an account
    Requires a username, password, email, and birthdate
  • Single Movie View- Shows all data about a single movie
  • Genre View- Shows all data about a specific genre
  • Director View- Shows all data about a specific director
  • User Profile View- Shows user’s details and list of favorite movies

After building all the views and connecting the interface to the API endpoints I had a functional app, however, to make this app more scalable in the future Redux was implemented to control the state of the app and the data displayed. In finishing up the project the app’s styling was touched up to provide a consistent and appealing user interface

Challenge

It was a lot to take in all at once! Having experience with databases made working with MongoDb pretty straightforward and styling has always been fairly easy for me. However understanding React was very challenging and I still don’t fully understand it. I found myself constantly having to rely on provided materials such as the example code just to get the app to function correctly. After completing this project I realized I needed to continue working through tutorials and the documentation to get a better understanding of React. I have found a few tutorials that incorporate my love for making music and learning React. I plan to start working through these tutorials and create virtual music instruments

Duration

This project took approximately 6 weeks to complete and was done ahead of schedule. In retrospect, I should have taken more time to fully understand the basics of React
